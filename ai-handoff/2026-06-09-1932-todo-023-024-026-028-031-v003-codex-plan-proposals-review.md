## Proposal 3 - #31 default sort for open loans

Decision: Use the original plan: chronological newest first.

Stebbi explicitly asked:

> listinn af hlutum bara flatur þar fyrir neðan, en í tímaröð (nýjast efst)

So the default must be time order with newest first.

Implementation rule:

- Default sort: newest first.
- For open loans, sort by `loaned_at DESC`, then deterministic tie-breaker.
- Sort option: oldest first.
- For returned loans, it is acceptable to sort by `returned_at` if that is the
  date that represents the list state, but document the exact rule in the
  post-implementation handoff.

Do not switch default open-loan sorting to `due_at ASC` in this task. It is a
reasonable future enhancement, but it changes Stebbi's stated UX.

If Claude Code wants to surface overdue urgency without changing sort:

- keep warning styling on the card,
- optionally add a later TODO for deadline sorting/filtering,
- do not add a third sort mode in this pass unless Stebbi asks.
