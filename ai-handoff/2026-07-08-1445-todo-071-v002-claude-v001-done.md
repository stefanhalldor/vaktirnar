## Problem solved

Forecast-point distance from road was gated behind `hasSeparateForecastPoint` and appeared late in the card, after weather values. The explainer section was hard to find (small muted text-link) and had redundant copy (short sentence above, same long text inside the drawer).

## What changed

### `components/weather/TravelAuditMap.tsx` — PointDetailsPanel

**Moved and ungated the distance line.**

Before: the distance text was inside `{summary.hasSeparateForecastPoint && (...)}` in position 7, after weather values.

After: a new section 5 renders the distance unconditionally (no `hasSeparateForecastPoint` gate), positioned between ETA (4) and forecast time (6):

```tsx
{/* 5. Forecast point distance from road */}
<span>
  {summary.forecastDistanceFromRouteM < 50
    ? tf('forecastPointOnRoute')
    : summary.forecastDistanceFromRouteM < 1000
    ? tf('forecastPointDistanceMeters', { meters: summary.forecastDistanceFromRouteM })
    : tf('forecastPointDistanceKilometers', { kilometers: formatNum(summary.forecastDistanceFromRouteM / 1000, locale) })}
</span>
```

The old section 7 now only retains the coord line (`forecastPointCoord`) inside `hasSeparateForecastPoint`, and the placeLabel / OSM attribution — the redundant distance text was removed from there.

Card info order is now:
1. Semantic title (`Mest krefjandi á leiðinni` or `Valinn punktur`)
2. `Punktur x/y`
3. `Brottfarartími: kl. ...`
4. `Áætlaður tími N km frá ...: kl. ...`
5. `Spápunktur um X m frá veginum.` (always shown)
6. `Veðurspá á þessum stað kl. ...`
7. Weather values (`Vindur · Úrkoma · Hiti`)
8. `Spágildi notað: kl. ...` (if forecast hour differs from ETA)
9. Forecast point coord + place label (when applicable)
10. Links (`Skoða veðurspá`, `Opna á korti`, `Hrá met.no gögn`)

### `app/auth-mvp/vedrid/FerdalagidClient.tsx` — explainer section

- The section now has `border-t border-border pt-3 mt-1` on the outer wrapper for clearer visual separation.
- `howAssessedShort` (now contains the long body text) is shown always above the button with `leading-relaxed`.
- The button title uses `text-sm font-medium text-foreground` instead of `text-xs text-muted-foreground` — much more visible.
- Icon size bumped from 12 to 14 to match the larger text.
- The redundant `howAssessedBody` paragraph inside the drawer was removed entirely. The drawer now only shows `betaTransparencyCopy` + `RoutePointRow` list.
