## Staðfesting á P1 galllanum

Codex er rétt.

Í `lib/weather/trip-assessment.ts` er núverandi kóði:

```ts
const status = worstStatus(allStatuses.length > 0 ? allStatuses : ['gult'])
```

`allStatuses` inniheldur aðeins niðurstöður frá `assessRouteLeg()` — og 'gult' ef leg input vantar.
Structural validation issues (unknown_from_stop, unknown_to_stop, non_adjacent_leg,
single_drive_requires_one_leg) bæta **ekkert** við `allStatuses`.

Þ.e.a.s. þetta er möguleg niðurstaða í dag:

```ts
{ status: 'graent', validationIssues: ['unknown_from_stop'] }
```

Þetta brýtur documentationina sem segir að malformed trips "contribute 'gult' to the aggregate".

Núverandi próf í `weather-trip-assessment.test.ts` athuga hvort validationIssues eru til
staðar — en skoða **ekki** hvort status er non-green þegar structural issue er til staðar.

## Sammála tillögu Codex

Tillaga Codex er rétt og einföld:

```ts
const aggregateStatuses = validationIssues.length > 0
  ? [...allStatuses, 'gult' as WeatherStatus]
  : allStatuses

const status = worstStatus(aggregateStatuses.length > 0 ? aggregateStatuses : ['gult'])
```

Þetta tryggir:
- valid green trip -> `graent`
- valid yellow trip -> `gult`
- valid red trip -> `rautt`
- invalid green trip -> `gult`
- invalid red trip -> `rautt` (rautt sigar yfir gult)
- no legs/no statuses -> `gult`

`worstLegId` er óbreytt — miðast áfram við leg assessments eingöngu.

## Próf sem þarf að bæta við / styrkja

Eins og Codex lýsir:

1. `unknown_from_stop` + green leg input -> `status: 'gult'`
2. `unknown_to_stop` + green leg input -> `status: 'gult'`
3. `single_drive_requires_one_leg` + two green legs -> `status: 'gult'`
4. Nýtt próf: `non_adjacent_leg` + green legs -> `validationIssues` contains `non_adjacent_leg`, `status: 'gult'`
5. Red precedence: structural issue + red leg -> `status: 'rautt'` (ekki downgraded til `gult`)
