### P1: Precipitation threshold -- STAÐFEST, LJÓST

`dry.maxPrecipMmPerHour = 0.1` er notað á þremur stöðum í `travel.ts` (línur 87 og 94) og einnig í `tools.ts` (línur 19, 213, 300) fyrir golf og grill. Ef þetta yrði breytt globally myndi 0.2 mm/h gera golftíma gult. Þetta er rangt.

Leiðrétting er nákvæmlega eins og Codex lýsir: bæta við `travel` hluta í `thresholds.ts` án þess að snerta `dry`. Þetta er sérstaklega einangrað.

```ts
travel: {
  cautionPrecipMmPerHour: 1.0,
  heavyPrecipMmPerHour: 3.0,
},
```

Í `evalDrivingLeg` (travel.ts:87, 94): skipta `WEATHER_THRESHOLDS.dry.maxPrecipMmPerHour` út fyrir `WEATHER_THRESHOLDS.travel.cautionPrecipMmPerHour`.

`reasonToText('precipitation')` (travel.ts:357) skilar nú `'Mikil úrkoma'` -- ætti að vera `'Rigning á leiðinni'` þar sem `'precipitation'` reason code merkir 1.0-3.0 mm/h. Heavy precipitation (>= 3.0) er hægt að koma inn síðar sem `heavy_precipitation` reason code ef þess þarf; enn sem komið er er nóg að breyta textanum.

### P1: Forecast parser next_6_hours fallback -- STAÐFEST, AUÐVELDUR FIX

Kóðinn á línu 44 í `forecast.ts`:

```ts
const precipitation = next1?.details?.precipitation_amount ?? next6?.details?.precipitation_amount ?? 0
```

`next_6_hours.precipitation_amount` er heildarmagn yfir 6 klst. Ef þetta er geymt sem `precipitationMmPerHour` án deilingar er það 6x of hátt. 6.0 mm yfir 6 klst = 1.0 mm/klst.

Leiðrétting:

```ts
const next1Precip = next1?.details?.precipitation_amount
const next6Precip = next6?.details?.precipitation_amount
const precipitation = next1Precip !== undefined
  ? next1Precip
  : next6Precip !== undefined
    ? next6Precip / 6
    : 0
```

Þetta hefur engin áhrif á `dry.maxPrecipMmPerHour` eða golf/grill -- það dregur bara úr ofmati þegar `next_1_hours` gögn eru ekki til staðar (t.d. spár langt fram í tímann eða sumar tímapunktar met.no).

Athugasemd: met.no skilar venjulega `next_1_hours` fyrir næstu 66 klst og `next_6_hours` fyrir tímapunkta þar á eftir. Flestir route-assessment tímapunktar falla í 48h glugganum þar sem `next_1_hours` er til staðar. Deilingin á `next_6_hours / 6` leiðréttir outliers en hefur lítil áhrif á nýlegar spár.
