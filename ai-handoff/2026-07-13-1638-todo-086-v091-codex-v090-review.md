- Each page query uses:
  - `.order('station_id')`
  - `.order('forecast_time')`
  - `.range(from, from + PAGE_SIZE - 1)`
- If a later page fails, the function keeps fail-open behavior and maps rows fetched so far.
- New tests cover:
  - fetching a second page when the first page has exactly 1000 rows
  - preserving page 1 and page 2 rows
  - stopping after a short page
  - partial map on mid-pagination error
  - no throw on DB errors
  - ok/stale/unavailable mapping
  - field mapping
  - empty station ID list

This is a good scoped fix. It does not touch SQL, Supabase schema, routes, UI, feature gates, auth, RLS, or production data.
