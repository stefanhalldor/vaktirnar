## What was done

Full execution of the v104 plan: always-visible timeline scrubber in single-departure mode + status filter chips.

### 1. Single-departure timeline (travel.ts + types.ts)

Replaced `findNextCaution` with `buildSingleDepartureTimeline`. Key difference:

- Old: called only when outbound was green, returned one `NextCaution`
- New: always called in single-departure mode, returns both `timelineCandidates` and `nextCaution`

`timelineCandidates` is hourly from earliest departure to `min(coverage cap, 48h)`. Each candidate has `pointStatuses` for map coloring. `timelineCandidates[0]` = current departure.

`nextCaution` is derived from `timelineCandidates[i > 0].find(c => c.status !== 'graent' && c.reasonCode !== 'no_data')` — no duplicate logic. Behavior is identical to before when outbound is green. Undefined when outbound is not green (no change).

New field in `TravelPlan.outbound`:
```ts
timelineCandidates?: TravelCandidate[]
```

### 2. Status filter chips (DepartureHeatmap.tsx)

New internal state: `hiddenStatuses: Set<SlotStatus>`. Default = nothing hidden (all visible).

Filter chip row appears only when non-green slots exist. Shows:
- "Allt" chip — resets to all visible; active when nothing hidden
- Per-status chips with count (e.g. "Varúð (4)") — toggle visibility

When a status is hidden and the currently selected slot has that status, the slot is deselected (`onSelectIdx(null)`).

Empty state when all slots are filtered out: `timelineEmptyFilter` text + "Sýna allt" reset button.

Filters reset to all-visible when `candidates` changes (new result loaded).

Legend row removed — filter chips replace it when non-green exists. When all-green, no legend needed (dots are self-explanatory).

### 3. Always-visible scrubber in single-departure mode (FerdalagidClient.tsx)

Added `outboundDisplayCandidates`:
```ts
const outboundDisplayCandidates = result?.travelPlan?.outbound.windowMode
  ? outboundCandidates
  : (result?.travelPlan?.outbound.timelineCandidates ?? [])
```

Outbound heatmap condition changed from `windowMode && candidates.length > 1` to `outboundDisplayCandidates.length > 1`. This means the scrubber now appears for single-departure results (as long as forecast coverage allows >1 hourly slot).

Single-departure mode passes `title={tf('timelineSingleDepartureTitle')}` to the scrubber. Window mode passes `undefined` (defaults to existing "Hvenær er best að leggja af stað?").

`heatmapHighlightedIssue` and `selectedCandidatePointStatuses` both updated to use `outboundDisplayCandidates` instead of `outboundCandidates`.

### 4. Layout reorder: map before scrubber (FerdalagidClient.tsx)

Result card → interactive map → outbound scrubber → return scrubber → explainer.

Previously the scrubber was above the map. Now it sits below as Stebbi described ("scrubber neðst").
