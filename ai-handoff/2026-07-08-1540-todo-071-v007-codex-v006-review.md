## Findings

### Medium - The original `Engin gögn` confusion remains in the selected departure summary

Claude Code verified that `SlotDetail` is technically using a different aggregate window from the individual point rows, and that `Engin gögn` can therefore be technically correct. However, Stebbi's issue was not only data correctness; it was that the UI now says `Engin gögn` prominently while the expanded points visibly contain weather data.

The unchanged code still renders only the header plus `Engin gögn` for no-data departure slots:

- `components/weather/DepartureHeatmap.tsx:290-294`

That means the confusing product state is still visible to users. Codex recommends a tiny copy follow-up, not an algorithm change:

- replace or supplement the selected-slot line with a more precise message like `Ekki nóg gögn til að meta þennan brottfarartíma.`;
- keep true no-data behavior;
- put the new copy in `messages/is.json` and `messages/en.json`;
- do not change route/weather thresholds.

This would explain why the departure slot cannot be classified even if route-point forecast rows below have partial/default-window data.

### Low - Point-card status is visible only as color for sighted users

`RoutePointRow` adds a colored left border and dot, and the status label is available to screen readers:

- `app/auth-mvp/vedrid/FerdalagidClient.tsx:1147-1169`

That is a good start, but `Design.md` says status colors must not be the only meaning. For sighted users, especially color-blind users, the row status is still only the color/dot. The visible text values help, but the status category itself (`Gott veður`, `Óþægilegt`, `Hættulegt`, `Engin gögn`) is not shown unless the row has no `summaryForWindow`.

Codex recommends adding a very small visible status badge/text next to `Punktur x/y`, using the same `statusLabel`, while keeping the `sr-only` label or using one visible label that also works for assistive tech. It can be compact enough not to clutter the long list.

### Low - Status colors are based on `summaryForWindow`, not the selected departure slot

The handoff documents that individual rows use `pt.summaryForWindow`, while a selected departure slot may use a different candidate window. That is acceptable if the list is understood as the general/default route-point list. It does mean the row colors may not visually explain the selected `Engin gögn` slot.

This is not a blocker, but the copy fix above becomes more important because it tells users the two sections are not evaluating exactly the same window.
