## Findings

### Medium - `RoutePointRow` under `Allir spápunktarnir á leiðinni` still uses the old sparse layout

The main map/detail card in `TravelAuditMap.PointDetailsPanel` was updated correctly, but the actual all-points list under the renamed explainer still renders via `RoutePointRow` in `app/auth-mvp/vedrid/FerdalagidClient.tsx:1139`.

That row still only shows:

- `Punktur x/y`
- distance from origin as `km frá uppruna`
- wind/gust/precipitation when `summaryForWindow` exists
- raw met.no coordinate
- the three links

It does not show the requested shared/full structure:

- `Brottfarartími: kl. ...`
- `Áætlaður tími N km frá ...: kl. ...`
- `Spápunktur um X m frá veginum.`
- `Veðurspá á þessum stað kl. ...`
- temperature

This means v002 only partially satisfies Stebbi's request:

> "þessi framsetning sé einnig notuð á öllum detail spjöldunum undir 'Hvernig er þetta metið?'"

Suggested fix:

- Update `RoutePointRow` to use the same point metadata order as `PointDetailsPanel`, as far as its data allows.
- `RouteWeatherPoint.summaryForWindow` already has `etaIso`, `forecastTimeIso`, `decisiveTempC`, wind/gust/precip values.
- Forecast-point distance can be derived the same way as `TravelAuditMap` derives `forecastDistanceFromRouteM` from route coordinate vs forecast coordinate.
- If departure time is not directly available in `RouteWeatherPoint`, either:
  - pass the active/default candidate departure into the row list, or
  - omit only that line in all-points rows with a conscious product note.

This is not a security issue, but it is an acceptance gap for TODO-071.

### Low - Small-distance wording may not match Stebbi's requested numeric line

`components/weather/TravelAuditMap.tsx:685-692` keeps the old `< 50m` branch:

```tsx

  ? tf('forecastPointOnRoute')
  : ...
```

So some cards will say `Spápunktur er nánast á veginum.` instead of `Spápunktur um X m frá veginum.`

This may be fine if Stebbi likes the human wording for tiny distances, but the latest request specifically asked to get the road-distance line back as `Spápunktur um X m frá veginum.`. If Stebbi wants literal consistency, remove the `<50m` wording branch or change `forecastPointOnRoute` to include a rounded meter value.

Not a blocker if Stebbi accepts the current wording.
