## Findings

### Medium - `SHORTER_DISTANCE` can be auto-selected without being visibly marked as shorter/experimental

Current code always adds `requestedReferenceRoutes: ["SHORTER_DISTANCE"]` in `lib/weather/google.server.ts`, sorts route options by `durationS`, and `FerdalagidClient` auto-selects `options[0]`.

Relevant code:

- `lib/weather/google.server.ts:167` adds `requestedReferenceRoutes`.
- `lib/weather/google.server.ts:236-237` sorts by fastest duration.
- `app/auth-mvp/vedrid/FerdalagidClient.tsx:184-186` auto-selects the first option.
- `components/weather/RouteSelectionStep.tsx:451-454` labels `idx === 0` as `Fljótlegasta leið` before checking `SHORTER_DISTANCE`.

If Google's shorter-distance route is also the fastest route, which is plausible for Garðabær -> Þorlákshöfn, it will be auto-selected and shown as `Fljótlegasta leið`, not `Styttri leið`. That makes the experimental/pre-GA route objective invisible exactly when it matters most.

Recommended before broad ship:

- Keep fastest sorting and auto-select if Stebbi wants that behavior.
- But visibly preserve the `SHORTER_DISTANCE` designation when labels include it, even for `idx === 0`.
- Example options:
  - title `Fljótlegasta leið` plus small secondary text/badge `Styttri leið`,
  - or title `Styttri leið` plus secondary text `Fljótlegust`.

This is not a blocker for localhost validation, but it is a product/safety issue before production rollout because Google documents shorter-distance routes as Experimental/pre-GA and notes they may choose less comfortable legal paths such as local roads, dirt roads, or parking lots.

### Low - v198 localhost check may mislead Stebbi about where the shorter route appears

v198 says to expect `Styttri leið` as a second option if Google returns Route 39. With the current sort/label logic, if Route 39 is faster than Route 427, it should be first and labelled `Fljótlegasta leið`.

For localhost validation, Stebbi should inspect labels in the terminal diagnostics, route description, distance, duration, and map path, not only whether a second card says `Styttri leið`.

### Low - Missing explicit final-submit regression test for selected `SHORTER_DISTANCE`

The provider tests are good and cover:

- request body includes `SHORTER_DISTANCE`,
- routeToken field mask,
- sorting,
- distinct shorter-distance route,
- deduped matching geometry with merged labels.

The v197 request also asked for a selected `SHORTER_DISTANCE` route to survive final submit recomputation. I did not see a new API-level test for that specific path. This is probably low risk because final submit calls the same `provider.getRouteOptions` and matches the same geometry-based route id, but it is still worth adding if this moves from localhost experiment toward release.
