**`selectStationRef`** — ref sem heldur alltaf nýjasta selection-fallinu, leysir stale closure í Google Maps marker listeners:
```tsx
const selectStationRef = useRef<(id: string) => void>(() => {})
selectStationRef.current = (stationId: string) => {
  const newId = selectedId === stationId ? null : stationId
  setSelectedId(newId)
  const url = new URL(window.location.href)
  if (newId) url.searchParams.set('stationId', newId)
  else url.searchParams.delete('stationId')
  router.replace(url.pathname + url.search, { scroll: false })
}
```

**Marker click:** `setSelectedId(prev => ...)` → `selectStationRef.current(stations[idx].stationId)`

**List click:** `setSelectedId(prev => ...)` → `selectStationRef.current(station.stationId)`

**URL init eftir data load:** les `searchParams.get('stationId')` og setur `selectedId` ef stöðin er í registry:

### `app/auth-mvp/vedrid/puls/stod/[stationId]/VedurstofanPulsClient.tsx`

Bætti við `returnTo: string | null` prop og `resolveBackHref` validation fallið:

```ts
function resolveBackHref(returnTo: string | null, stationId: string): string {
  const fallback = `/auth-mvp/vedrid/elta-vedrid?stationId=${stationId}`
  if (!returnTo) return fallback
  try {
    const decoded = decodeURIComponent(returnTo)
    if (decoded.startsWith('/auth-mvp/vedrid')) return decoded
  } catch { /* ignore malformed encoding */ }
  return fallback
}
```

Reglur:
- `returnTo=null` → fallback með `?stationId=`
- `returnTo` sem byrjar á `/auth-mvp/vedrid` → notað beint
- `returnTo=https://evil.example` → fallback (ytra URL hafnað)
- Malformed encoding → fallback

Back link notar `backHref` í stað hardcoded `/auth-mvp/vedrid/elta-vedrid`.
