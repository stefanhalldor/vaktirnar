## Implementation-level athugasemdir (engar krefjast Stebbi-svars)

### 1. `pointForecasts` verður breaking change

WorstMetric krefst `lat`, `lon`, `routeIndex`, `distanceFromOriginM` per route point. Núverandi type:

```ts
pointForecasts: Array<{ hours: HourPoint[] }>
```

Þarf að verða:

```ts
pointForecasts: Array<{ hours: HourPoint[]; lat: number; lon: number; routeIndex: number; distanceFromOriginM: number }>
```

Þetta breytir **öllum** tests sem búa til pointForecasts handvirkt — þ.e. nær allt `weather-travel.test.ts`. Claude Code þarf að bæta lat/lon/index/distance við hvert test-forecast object. Raungildi skipta ekki máli í prófum (t.d. `lat: 64.0, lon: -22.0, routeIndex: 0, distanceFromOriginM: 0`) en þau verða að vera til staðar vegna TypeScript.

Þetta er þekkt áhætta, ekki blocker — en stærsti hluti af test-uppfærslunni.

### 2. `distanceFromOriginM` approximation

Route geometry frá Google gefur bara lat/lon punkta, ekki cumulative distance per point. Claude Code getur notað eina af tveimur nálgunum:

- **Linear**: `routeIndex / (totalPoints - 1) * distanceM` — einfalt, getur verið misleidandi á beygjóttum leiðum.
- **Haversine cumulative**: leggur saman Haversine-fjarlægð milli samliggjandi punkta — nákvæmara, ~10 línur af kóða.

Mælt: Haversine cumulative — það er ekki flókið og gefur mun trúverðugri "62 km frá Reykjavík" í result texta. Claude Code ákveður við framkvæmd.

### 3. Lodging tests í `weather-travel.test.ts`

Núverandi lodging/stay tests munu brotna þegar `TravelWeatherInput` breytist (fjarlægar `returnDepartureAt`, lodgingKind verður ónotað í þessum pass). Þær geta annað hvort:

a. Verið fjarlægðar (clean slate).
b. Verið commented out / skipt yfir í `it.skip(...)` þar til lodging pass kemur.

Claude Code mælir með b — `it.skip` — þannig að test-intent heldur sér. Þetta þarf ekki Stebbi-samþykki.

## Status labels — shared keys

`statusGraent/statusGult/statusRautt` eru undir `teskeid.vedrid` í messages.json. Þær eru notaðar í tveimur stöðum:

- `FerdalagidClient.tsx` — þetta er Ferðalagið
- `VedridClient.tsx` — þetta er gamla /ask chat UI sem er enn til (falið en virkt)

v060 segir: "either confirm those surfaces are hidden or add travel-specific labels."

**Staðfesting:** VedridClient.tsx er falinn notendum (hann er ekki renderar úr `/auth-mvp/vedrid/page.tsx` lengur — FerdalagidClient tók við). Þannig að breyta `statusGult` í shared key frá "Meðgótt" í "Varúð" mun hafa áhrif á VedridClient ef hann væri sýnilegur, en þar sem hann er falinn er breytingin örugg.

Claude Code mun nota shared keys í þessum pass (ekki bæta við travel-specific under ferdalagid). Ef VedridClient þarf að koma aftur, þarf að sérsniðin labels þar — en það er framtíðarvandamál.
