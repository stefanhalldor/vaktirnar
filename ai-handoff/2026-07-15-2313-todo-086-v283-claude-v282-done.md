### 1. `lib/chat/repository.server.ts` — getFeedMessages first-name fix

`getFeedMessages` notaði `profileMap.get(row.user_id) ?? null` beint án `toPublicFirstName`. Þetta lak fullt `display_name` í gegnum Safnpúls.

```ts
authorName: row.user_id ? toPublicFirstName(profileMap.get(row.user_id) ?? null) : null,
```

- `toMessageDto` (per-thread skilaboð)

- `getPreviewMessages` (public preview)

### 5. `WeatherPulseSummary` — simplified composer UX

Fjarlægt gamla "Skrifa umferðarfrétt" hnappinn og toggle-logic.

Nýtt hegðunarmynstur:
- `canPost=false` (notendur án aðgangs): eingöngu preview + "Opna púlsinn" tengill. Enginn input.
- `canPost=true` (notendur með aðgang): input sýnilegur strax, engin hnappur til að opna.
- Thread init er lazy: POST /thread fer aðeins út þegar notandi smellir á Senda. Að skoða stöðvar skapar engar tómar threads.
- Eftir sendingu: input tæmist, preview refreshast sjálfkrafa.

```tsx
{canPost && (
  <div className="flex flex-col gap-1.5">
    <div className="flex gap-1.5">
      <input ... placeholder={t('pulseInputPlaceholder')} />
      <button ... disabled={sending || !composeBody.trim()}>
        {sending ? '...' : t('pulseSend')}
      </button>
    </div>
    {sendError && <p ...>{t('pulseSendError')}</p>}
  </div>
)}
```
