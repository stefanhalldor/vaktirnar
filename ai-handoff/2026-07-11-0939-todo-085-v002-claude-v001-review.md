## Tillögur að implementation plan

### 1. Ný pure function — wind distance classifier

Ný skrá eða viðbót við `lib/weather/assessment.ts`:

```ts
export type WindDistanceLabel =
  | 'innan-marka'
  | 'nalgast-othaegindi'
  | 'othaegilegt'
  | 'nalgast-haettumork'
  | 'haettulegt'

/**
 * Returns a fine-grained wind distance label for display purposes.
 * Internal WeatherStatus (graent/gult/rautt) remains unchanged.
 */
export function classifyWindDistance(
  wind: number,
  uncomfortableWindMs: number,
  dangerousWindMs: number,
): WindDistanceLabel {
  if (wind >= dangerousWindMs) return 'haettulegt'
  if (dangerousWindMs - wind < 2) return 'nalgast-haettumork'
  if (wind >= uncomfortableWindMs) return 'othaegilegt'
  if (uncomfortableWindMs - wind < 2) return 'nalgast-othaegindi'
  return 'innan-marka'
}
```

Internal mapping til WeatherStatus:
- `haettulegt` → rautt
- `nalgast-haettumork` → gult (með sérstakt viðvörunarlit í UI)
- `othaegilegt` → gult
- `nalgast-othaegindi` → gult
- `innan-marka` → graent

"distinct warning treatment". Er viðeigandi:

Mæling: Stebbi ákveður viðmótslitinn, Claude Code útfærir.

Þetta orðalag kemur fram á mörgum stöðum:

Er þetta hluti af #85 eða aðskilið? Mæling: gera það hluta af #85 þar sem

Hvað þýðir það í practice? Precipitation í forecast drawer og route point detail
má vera áfram — en precipitation threshold triggers í result status/pills hverfa.
