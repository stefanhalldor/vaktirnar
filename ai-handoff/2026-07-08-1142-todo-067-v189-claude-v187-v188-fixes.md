## What was done

### High — Route ID no longer includes traffic-aware duration

**`lib/weather/google.server.ts`**

Old fingerprint (fragile with TRAFFIC_AWARE):
```ts
`${route.distanceMeters}-${durationS}-${first[1].toFixed(4)},${first[0].toFixed(4)}-${last[1].toFixed(4)},${last[0].toFixed(4)}`
```

New fingerprint (geometry only — distance + first, middle, last coords):
```ts
`${route.distanceMeters}-${first[1].toFixed(4)},${first[0].toFixed(4)}-${mid[1].toFixed(4)},${mid[0].toFixed(4)}-${last[1].toFixed(4)},${last[0].toFixed(4)}`
```

`durationS` is completely removed from the ID. Traffic-aware duration changes between the route-step fetch and the final travel submit no longer cause `selected_route_unavailable`.

`durationS` is still stored on `RouteOption` for display and sorting — just not for identity.

### Medium — placeId validated and normalized at both route API boundaries

Added `normalizeOptionalPlaceId(raw: unknown): string | undefined` to:
- `app/api/teskeid/weather/travel/routes/route.ts`
- `app/api/teskeid/weather/travel/route.ts`

Normalizer:
- Rejects non-string: returns `undefined`
- Rejects empty string / whitespace-only: returns `undefined`
- Rejects strings > 500 chars: returns `undefined`
- Otherwise trims and returns

Both `originCandidate.placeId` and `destCandidate.placeId` now use `normalizeOptionalPlaceId(origin.placeId) ?? 'confirmed'` instead of `origin.placeId ?? 'confirmed'`. Malformed or oversized client-supplied placeIds cannot reach the Google Routes call.

### Low — Route description no longer truncated

`components/weather/RouteSelectionStep.tsx`: removed `truncate` from the description span so the full `via Þrengslavegur/Route 39` text is visible without needing to hover or read an aria-label.

### Diagnostic logging (dev only)

`lib/weather/google.server.ts` `getRouteOptions` now logs to the Next.js server terminal when `NODE_ENV !== 'production'`:

```json
[weather/google] getRouteOptions diagnostics: {
  "originType": "placeId",
  "destType": "placeId",
  "routeCount": 1,
  "routes": [
    {
      "distanceMeters": 67000,
      "durationS": 3480,
      "labels": ["DEFAULT_ROUTE"],
      "description": "Krýsuvíkurvegur og Suðurstrandarvegur/Leið 427"
    }
  ]
}
```

This tells Stebbi:
- whether Place IDs are being sent to Google (`originType`/`destType`)
- exactly how many routes Google returned
- the labels, descriptions, distance and duration for each route

No API keys, auth tokens, or user-identifying data are logged.
