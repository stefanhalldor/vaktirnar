### Finding 2 (Medium) - SQL63 er oklartt þegar fleiri en eitt pending bod er til

**`sql/63_switch_loan_role.sql`:**

- Bætt `v_pending_count integer` vid i DECLARE blokk
- Bætt `invalid_state` vid i status values skjal
- Bætt guard vid a undan Step 2 fallback:

```sql
IF v_inv_id IS NULL THEN
  SELECT COUNT(*) INTO v_pending_count
  FROM public.loan_invitations
  WHERE loan_id = p_loan_id
    AND status  = 'pending';

  IF v_pending_count > 1 THEN
    RETURN QUERY SELECT 'invalid_state'::text, NULL::text, NULL::uuid, NULL::uuid[];
    RETURN;
  END IF;

  SELECT inv.id INTO v_inv_id ...
END IF;
```

Ef actor er pending recipient (Step 1 fannd valid_inv_id), er guardinn sleppt - aðeins actual party fallback er varinn.
