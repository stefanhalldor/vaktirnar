## What was done

Two themes from Codex v117: MVP step order (remove `times`) and wizard stepper navigation.

### 1. P1 - `times` removed from MVP flow

**`FerdalagidClient.tsx`:**
- `WizardStep` type: removed `'times'`
- `TimeConstraintKind` type: removed entirely
- `STEP_ORDER`: `['route', 'trailer', 'thresholds', 'result']` (was `['route', 'times', 'trailer', 'thresholds', 'result']`)
- Removed state vars: `timeConstraintKind`, `timeConstraintValue`
- Removed `localToIso` helper function
- Removed `timeConstraintOptions`, `timeInputLabel` computed values
- `handleSubmit`: no longer sends `earliestDepartureAt`, `latestArrivalBy`, or `latestHomeBy` — body now only has `origin`, `destination`, `trailerKind`, `thresholdOverrides`
- `startOver`: no longer resets time state
- Removed the `times` JSX step block (~50 lines of segmented control + datetime input)
- Assumptions step: removed the time `AssumptionRow` (was showing kind + value)

Server-side time field support is unchanged. The API still accepts and validates all three time fields for future advanced use, but the MVP client never sends them.

**Result copy:** No server-side changes needed. Since no time fields are sent, `checkTravelWeather` always runs in window mode. The "Ferðin frá kl. HH:MM..." branch in `travel.ts` is only reached when there is an explicit departure time. Without one the server produces window-mode copy ("Ferðaveður lítur vel út. Besti brottfararglugginn..."), which does not imply a user-selected time.

### 2. P2 - Wizard stepper navigation added

Added a `<nav>` stepper under the header for the 4 MVP steps (hidden on the `assumptions` side-step since it is not in `STEP_ORDER`).

**Icons (lucide-react):** `Route`, `Truck`, `SlidersHorizontal`, `CheckCircle2` — added to import.

**Stepper behavior:**
- Current step: `text-primary` + a small filled dot below the label
- Completed steps: `text-primary/50`, clickable, navigates directly (clears `returnToStep`)
- Future steps: `text-muted-foreground/30`, `disabled`, pointer unchanged
- `aria-current="step"` on active step for accessibility

**Labels** (all via i18n, no hardcoded text):
- IS: Leið / Eftirvagn / Veðurmörk / Niðurstaða
- EN: Route / Trailer / Thresholds / Result

**Layout:** 4 equal `flex-1` columns, `text-[10px]` labels with `truncate`. At 360px inner width each column is ~82px which fits all IS labels without overflow.

### 3. i18n keys added

Both `messages/is.json` and `messages/en.json` received 5 new keys under `ferdalagid`:
`stepNavAriaLabel`, `stepNavRoute`, `stepNavTrailer`, `stepNavThresholds`, `stepNavResult`.
