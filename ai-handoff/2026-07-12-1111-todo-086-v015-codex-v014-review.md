### P1 - Only verified station coordinates should enter the default list

For this project, I recommend being stricter than "ID confirmed, coords approx".

For the default `VEDURSTOFAN_STATIONS` list:

- Add a station only if both station ID and coordinates are verified from an official station page.
- Use `coordinatesVerified: true` for every default-list station added in this pass.
- Avoid adding approximate coordinates to the default list, because the mapper treats all stations equally today.

If we need a separate "candidate/unverified stations" scratch list later, keep it outside default route mapping.

### P2 - Expansion should be route-coverage focused, not "all stations"

The goal is not to scrape every Veðurstofa station. The useful first coverage set is route-weather coverage:

- capital area / Reykjanes / Route 41
- Hellisheiði / Suðurland
- Selfoss / Hveragerði area
- Vík / Kirkjubæjarklaustur / South coast
- Höfn / Southeast
- Egilsstaðir / East
- Akureyri / North
- common mountain/road-risk stations where available

This should be enough to make a later shadow comparison meaningful without bloating the list.

### P2 - Keep Phase 1C separate

Do not combine station-list expansion with fetch/cache wrapper work.

Recommended sequence:

1. Expand station list + tests.
2. Review.
3. Then Phase 1C fetch/cache wrapper.

This keeps failures easy to diagnose.
