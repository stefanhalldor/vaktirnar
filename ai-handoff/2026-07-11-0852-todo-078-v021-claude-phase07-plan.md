## Markmið Phase 0.7

Bæta við `WEATHER_TRIP_ENABLED` feature flag og sýna litla conversion affordance í
`/auth-mvp/vedrid` þegar notandi hefur fengið niðurstöðu á einstaka akstri.

Þetta er seed UI — ekkert í multi-stop flæðinu virkar ennþá. Notandi sér
mögulegann næsta skref en getur ekki farið þangað fyrr en Phase 0.8+.

Default: flagið er af. Engin ný UI sjáanleg í production án þess að kveikja.

## Hvað er gert í Phase 0.7

### 1. Feature flag — server component

`app/auth-mvp/vedrid/page.tsx` les `WEATHER_TRIP_ENABLED` og sendir sem prop:

```tsx
export default async function VedridPage() {
  const { user } = await guardTeskeidSession()
  await guardFeatureAccess(user.email!, 'vedrid')
  const tripEnabled = process.env.WEATHER_TRIP_ENABLED === 'true'
  return <FerdalagidClient tripEnabled={tripEnabled} />
}
```

Flagið er server-side only — aldrei `NEXT_PUBLIC_`. Sett í `.env.local` þegar
Stebbi vill prófa.

### 2. Prop til FerdalagidClient

```tsx
export function FerdalagidClient({
  isGuest = false,
  tripEnabled = false,
}: {
  isGuest?: boolean
  tripEnabled?: boolean
} = {}) { ... }
```

### 3. Conversion affordance í result step

Þegar `step === 'result'` og `result !== null` og `tripEnabled === true`:
sýna lítinn "Breyta í ferðalag" link/knapp neðst á result card, undir öðru
result efni.

Útlit (Design.md compliant):
- Ghost-style link eða `text-sm` textalink með `Route` icon (Lucide)
- Ekki floating button, ekki stór primary button
- Ekki nested card — inni í núverandi result section
- Litill, ekki truflandi
- Mobile-first: touch target minnst 40px, nægt bil

Þegar smellt er á hann: sýna inline "Ferðalag kemur fljótlega" placeholder texta
(eða disable knappinn með tooltip). Ekkert flæði opnast ennþá.

Mögulega útlit:

```
[ Route icon ]  Bæta við stigi →    (ghost textalink neðst)
```

Eða:

```
[ + ]  Breyta í ferðalag            (ghost button, disabled eða með 'beta' badge)
```

### 4. Messages

Bæta við í `messages/is.json` og `messages/en.json` undir
`teskeid.vedrid.ferdalagid`:

```json
"convertToTrip": "Breyta í ferðalag",
"tripComingSoon": "Ferðalag kemur fljótlega"
```

(Nákvæm lyklanöfn taka mið af existing messages-skipulagi.)

### 5. Engin breyting á WizardStep type

`WizardStep = 'route' | 'trailer' | 'thresholds' | 'result'` er óbreytt.
Engin 'trip' step bætt við ennþá.

## Design.md samræmi

- Mobile-first: affordance er inline, smá, ekki floating
- Ekki nested card: affordance er hluti af result section, ekki í nýjum card
- All text í messages: já, allar strengjar í is.json/en.json
- No dashboard sprawl: eitt lítið element, ekki nýr section
- Touch target: ghost link með padding, ≥40px
- Loading state: ekki við — affordance er static þegar result er til
- Engin animation: static element

## Óvissa / þarf að staðfesta

1. **Nákvæm staðsetning affordance** — hvar nákvæmlega á result screen kemur
   hann best: neðst á summary card, í footer area, eða öðruvísi. Codex eða
   Stebbi á að staðfesta áður en útfærsla hefst.

2. **Clicked behavior** — þegar notandi smellir á "Breyta í ferðalag":
   - Valkostur A: disabled knappinn + tooltip "Ferðalag kemur fljótlega"
   - Valkostur B: inline text sýndur í stað links eftir smelli
   - Valkostur C: ekkert (disabled frá byrjun)
   Codex ætti að velja. Valkostur A eða B er æskilegast — dæmi um það að sýna
   notandanum að hluturinn sé á leiðinni.

3. **Þarf að fara í Phase 0.7 í auth-mvp/vedrid eingöngu** eða líka í public
   `/vedrid`? Mæling: public `/vedrid` er til gesta, Ferðalag er meiri hugmynd
   — best að fara í auth-only fyrst.

4. **Nafnið á flaginu** — `WEATHER_TRIP_ENABLED` er Codex tillaga úr v010. Er
   það rétt nafn? Ætti það að heita `WEATHER_FERDALAG_ENABLED`?

## Localhost checks fyrir Stebbi eftir Phase 0.7

Þegar `WEATHER_TRIP_ENABLED` er ekki sett (eða `false`):
1. Reikna leið í `/auth-mvp/vedrid`
2. Fá niðurstöðu — engin "Breyta í ferðalag" UI sést
3. Allt aðra UI óbreytt

Þegar `WEATHER_TRIP_ENABLED=true` er sett í `.env.local`:
1. Reikna leið í `/auth-mvp/vedrid`
2. Fá niðurstöðu — sjá lítinn "Breyta í ferðalag" element
3. Smella á hann — sjá placeholder ("kemur fljótlega") eða disabled state
4. Önnur UI óbreytt
