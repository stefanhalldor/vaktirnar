## Hvað var gert

### Blocker 1 - SQL64 patch: ambiguous `status` fixed

**`sql/64_fix_switch_loan_role_ambiguous_status.sql`** - ný skrá:

- `DROP FUNCTION IF EXISTS public.switch_loan_role(uuid, uuid)`
- `CREATE OR REPLACE FUNCTION public.switch_loan_role(...)` med fix-udum COUNT queries

Vandinn: badar COUNT queries notudu ómerkt `loan_id` og `status` sem konfliktaði vid
output column `status` i RETURNS TABLE. Postgres kastar `42702: column reference "status" is ambiguous`.

Lagfæring: `FROM public.loan_invitations inv` alias baett vid og allar dálkavísur merktar `inv.loan_id`, `inv.status`.

```sql
-- Adur (villa):
FROM public.loan_invitations
WHERE loan_id = p_loan_id
  AND status  = 'pending';

-- Eftir (fix):
FROM public.loan_invitations inv
WHERE inv.loan_id = p_loan_id
  AND inv.status  = 'pending';
```

Sama fix i badum COUNT queries (Step 2 guard og actual-party guard).

**`sql/63_switch_loan_role.sql`** - baeda COUNT queries fixadar til samræmis vid SQL64 (fyrir ný environments).

**`lib/__tests__/sql-migration.test.ts`:**
