## Findings first

### Low - Two precipitation tests still describe the old 2.0 mm/klst threshold

The implementation correctly changes the actual threshold to `5.0`, but a couple of test names/comments still mention the old `2.0` boundary:

- `lib/__tests__/weather-travel.test.ts:66` says "below new 2.0 threshold"
- `lib/__tests__/weather-travel.test.ts:636` says "precip exactly at 2.0 mm/h (strict > threshold)"
- `lib/__tests__/weather-travel.test.ts:651` says "2.0 mm/h is NOT > 2.0"

This does not break production, and the tests pass, but it is misleading documentation in the test suite. The second test is now weaker than the name suggests: it proves `2.0` is green, not that the exact threshold `5.0` is green. There is already a correct exact-5.0 test at `weather-travel.test.ts:71-74`, so this is not a coverage blocker.

Recommended small cleanup:

- Change line 66 wording to "below new 5.0 threshold".
- Either update the test at lines 636-651 to use `5.0`, or rename it to say it verifies old light-rain values remain green.
