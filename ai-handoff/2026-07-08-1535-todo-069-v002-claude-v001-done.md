## Files created

### `sql/71_teskeid_usage_events.sql`
New table `public.teskeid_usage_events` with:
- `id`, `user_id` (FK to auth.users), `feature_key`, `event_name`, `path`, `metadata` (jsonb), `created_at`
- 4 check constraints: feature_key format, event_name format, path length, metadata must be jsonb object
- 4 indexes: by created_at, by feature+created, by event+created, by user+created (WHERE user_id IS NOT NULL)
- RLS enabled, no anon/authenticated grants, service_role gets SELECT + INSERT only
- **NOT run. Stebbi must run this explicitly.**

### `lib/teskeid/usage.server.ts`
Server-only helper (`import 'server-only'`):
- `sanitizeUsageMetadata(input)` — strips keys matching `/email|name|address|lat|lon|place|polyline|forecast|secret|token/i`, strings > 200 chars, nested objects. Exported for testing.
- `recordTeskeidUsageEvent(input)` — async, silently fails, logs `[usage] insert failed` on error, never throws
- `routePairFingerprint(origin, destination)` — HMAC-SHA256 of `lat.toFixed(3):lon.toFixed(3)->lat.toFixed(3):lon.toFixed(3)`. Uses `USAGE_EVENT_SECRET` env var (falls back to empty string until secret is set)

### `app/api/admin/teskeid-usage/route.ts`
GET endpoint:
- Auth: `createClient()` + `requireAdmin(supabase)` (401/403 on failure)
- Period param: same allowed periods as `/api/admin/analytics`, 400 on invalid
- Fetches from `teskeid_usage_events` via `getAdmin()`, aggregates in JS
- Response contains only aggregated counts — no user_id, no raw rows, no route pair hashes exposed
- `summary`: total_events, unique_users, active_features, weather_route_calculations, weather_distinct_route_pairs, weather_final_forecasts, weather_route_to_result_conversion
- `features`: array of 4 features (vedrid/minnid/tengsl/umonnun) with per-feature counts
- `weather`: route calc counts, failure counts, distinct pairs, final forecast counts, conversion rate, route count buckets, curated route label counts
- `events_over_time`: daily counts

## Files modified

### `app/api/teskeid/weather/travel/routes/route.ts`
Added `recordTeskeidUsageEvent` + `routePairFingerprint` imports.

Records events after auth/validation/provider call:
- `weather_route_options_failed` — when provider throws or returns 0 routes
- `weather_route_options_calculated` — on success, with metadata:
  - `routePairHash`, `provider`, `routeCount`
  - `originIdPresent`, `destinationIdPresent` (booleans)

All calls are fire-and-forget (`void`). Usage failure cannot change the response.
