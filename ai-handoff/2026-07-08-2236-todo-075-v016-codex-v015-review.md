## Findings

### P1 - "Á leiðinni" shows the forecast hour as if it were the route ETA

In `app/auth-mvp/vedrid/FerdalagidClient.tsx:892`, the new top summary uses:

`const timeLabel = dp ? formatKlTime(dp.forecastTimeIso) : ...`

That value is then rendered in `slotDetailWorstDistanceAt` at `app/auth-mvp/vedrid/FerdalagidClient.tsx:901`, which reads as:

`Mest krefjandi er X km frá ..., kl. HH:mm.`

For `displayPoint`, `forecastTimeIso` is the weather forecast bucket, not the estimated time the user reaches that route point. The existing detail card already computes the route ETA correctly from departure, arrival, and `dp.routeFraction` in `components/weather/DepartureHeatmap.tsx:324-338`.

Impact: the top box can say the most demanding point happens at e.g. `kl. 06:00` when the car reaches it at `kl. 06:05`. This is exactly the screenshot-proof context Stebbi wanted to improve, so I would not commit/push/deploy v015 unchanged.

Recommended fix: extract or duplicate the same ETA calculation used by `SlotDetail` and use that for the "Mest krefjandi er ... kl." line. Keep `forecastTimeIso` only for weather-forecast wording such as "Veðurspá á þessum stað kl. ...".

### P2 - Top summary passes the raw origin name into "frá {origin}"

`app/auth-mvp/vedrid/FerdalagidClient.tsx:901` passes `origin?.name ?? ''` into `slotDetailWorstDistanceAt`.

In Icelandic this likely produces text like:

`Mest krefjandi er 42 km frá Garðabær, kl. 21:40.`

The detail card avoids this by using `getOriginDisplay(...)` before passing the origin into the same translation (`components/weather/DepartureHeatmap.tsx:319`). The top summary should use the same dative/place-name handling, ideally via a shared helper instead of copying the mapping.

### P2 - Existing first-point edge case is still open in the new top card

The top summary only renders the "Mest krefjandi er ..." line when `distKm > 0` (`app/auth-mvp/vedrid/FerdalagidClient.tsx:899`). `SlotDetail` has the same shape at `components/weather/DepartureHeatmap.tsx:337`.

That leaves Stebbi's earlier TODO unresolved: if the most demanding point is the first point, the UI should say:

`Mest krefjandi er við upphaf ferðarinnar, kl. HH:mm.`

This does not have to block the sticky-day/date polish if Stebbi wants to keep it separate, but since v015 is already touching the same top summary text, it is a cheap and sensible fix in the same follow-up.

### P3 - Removed selected-slot label still has translation keys

`selectedSlotLabel` remains in `messages/is.json` and `messages/en.json` even though the visible "Valið:" line was removed. This has no user-facing impact, but it is dead copy and can be cleaned up during the next small pass.
