## Localhost QA checklist (for Stebbi)

Browser key needed: `NEXT_PUBLIC_GOOGLE_MAPS_BROWSER_KEY` (not `_API_KEY`).

1. Open `/auth-mvp/vedrid` and submit a route, e.g. Garðabær to Akureyri.
2. **Map loads**: interactive map appears (not static fallback), route line visible, colored weather point markers visible.
3. **No console errors**: DevTools shows no `Marker is not a constructor`, `LatLngBounds is not a constructor`, or similar library errors.
4. **Forecast markers**: for most weather points, a small gray outlined circle appears near or slightly offset from the colored route marker (met.no grid point). On flat coastal routes they may overlap exactly.
5. **Connector lines**: on routes through mountains or near coasts where the nearest road is far from the met.no grid, thin gray connector lines appear between route markers and forecast markers.
6. **Click markers**: clicking any route or forecast marker selects that point. Details panel updates.
7. **Details panel**: shows:
   - Point index / total
   - Distance from origin
   - Wind/gust/precip if available
   - "Punktur á leið: 64.XXXX, -22.XXXX"
   - "Spápunktur met.no: 64.XX, -22.XX" (if different from road coord)
   - Explanation text (if different)
   - Links: yr.no forecast, Google Maps, met.no raw
8. **Worst point**: the highlighted issue point is initially selected, shows red badge "Versti punktur".
9. **Second submit**: submit a different route. Map reinitializes cleanly (key={result.id} ensures unmount/remount).
10. **Green result**: if result is green and in single-departure mode, one of three lines appears below the main result text:
    - A future caution time and reason, or
    - "Engin varúð fannst..." with hours scanned, or
    - "Spágögn ná ekki..." (rare — only if forecast data is very sparse)
11. **Static fallback**: if browser key is missing or removed, the static map image appears (if present) or "Kortið náði ekki að hlaðast."
12. **Mobile**: check at 390px viewport. Map, details panel, and buttons should not overflow.

## Open question for Codex

1. **Forecast marker always vs. threshold**: currently forecast markers only show when forecast grid differs from road point by >50m. This means on flat coastal routes where met.no rounds identically, no forecast marker appears. Is this acceptable, or should we always show the outlined circle and just skip the connector? The details panel always shows both coords regardless.

2. **`nextCaution` in window mode**: not implemented. Scan would start after `latestDepartureIso + 1h`. Copy would need to say "eftir valda gluggann". Ready to implement if Stebbi wants it.

3. **`nextCaution.issue` on map**: the map currently ignores `nextCaution` entirely. If the current result is green (no `highlightedIssue`), the initial selected point falls back to destination or first point. Should the next-caution issue point be shown on the map as "Næsta varúð" instead? This would require adding a `nextCautionHighlighted` flag to `RouteWeatherPoint` — out of scope for this pass but easy to add.
