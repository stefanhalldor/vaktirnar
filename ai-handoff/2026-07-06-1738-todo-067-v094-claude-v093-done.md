### P1 — Heimferðar-fjarlægð reiknuð frá réttum enda

**`components/weather/travelAuditMap.helpers.ts`**

`candidateToIssue` fékk nýjan valfrjálsan þriðja parameter:
```typescript
opts?: { routeDistanceM?: number; legStartName?: string }
```

Innri hjálparfall `legStartDistance(distFromOrigin)`:
- `leg === 'return'` + `opts.routeDistanceM` gefið → skilar `routeDistanceM - distFromOrigin`
- Annars → skilar `distFromOrigin` óbreytt

Báðar return-statements í fallinu (precipitation og wind/gust) nota `legStartDistance` fyrir
`distanceFromLegStartM` og taka `legStartName: opts?.legStartName`.

**`components/weather/DepartureHeatmap.tsx`**

Bætti við `routeDistanceM?: number` og `leg?: 'outbound' | 'return'` props.

Í `SlotDetail`: `distKm` reiknaður sem:
```typescript
const legDist = rawDist !== undefined && leg === 'return' && routeDistanceM !== undefined
  ? routeDistanceM - rawDist
  : rawDist
```

`routeDistanceM` og `leg` eru sendar niður úr `DepartureHeatmap` í `SlotDetail`.

**`app/auth-mvp/vedrid/FerdalagidClient.tsx`**

- `routeDistanceM = Math.round(result.travelPlan!.route.distanceKm * 1000)` reiknað eitt sinn.
- Return `DepartureHeatmap` fær `leg="return"` og `routeDistanceM={routeDistanceM}`.
- `candidateToIssue` fyrir return-candidates fær `{ routeDistanceM, legStartName: destination?.name }`.
- `candidateToIssue` fyrir outbound-candidates fær `{ legStartName: origin?.name }`.

Niðurstaða: ef veðurpunktur er 84 km frá uppruna á 200 km leið, sýnir heimferðar-SlotDetail
"116 km frá [áfangastað]" í stað "84 km frá [áfangastað]".

### P1 — Grænir slotar hreinsa kortval

**`app/auth-mvp/vedrid/FerdalagidClient.tsx`**

`heatmapHighlightedIssue` fallback breytt:
```typescript
// Áður — grær slot gaf undefined:
? candidateToIssue(outboundCandidates[selectedHeatmapIdx], 'outbound')

// Nú — grær slot fallback á sjálfgefið highlighted issue:
? (candidateToIssue(outboundCandidates[selectedHeatmapIdx], 'outbound', ...) ?? result?.travelPlan?.highlightedIssue)
```

Sama mynstur fyrir return-leg.

Áhrif: þegar notandinn smellir á grænan slot, breytist `highlightedIssue` í
`result.travelPlan.highlightedIssue`. `TravelAuditMap` sync-effect keyrir og flytur
`selectedIndex` á sjálfgefna highlighted issue-punktinn. Kortið endar á réttum
sjálfgefnum punkti, ekki á fyrri rauðum/gulum slotpunkti.

Grænna slots-texti (`heatmapMapShowsDefault`) er enn sýndur þegar `selectedSlotIsGreen`
— hann útskýrir nú réttilega að kortið sýni sjálfgefna versta punktinn.

Copy leiðrétt: `veðrastöð` → `veðurpunkt` (veðurpunktur er nákvæmari lýsing á
líkanaðgerðum í kóðanum).
