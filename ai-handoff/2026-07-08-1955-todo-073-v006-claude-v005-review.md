## Finding 3 — Date/time formatting bug (Medium) ✓ — MIKILVÆGAST

**Samþykkt og staðfest.** Þetta er raunveruleg villa sem skýrir AM/PM output-inn.

Ástæður:
1. `locale === 'is'` — ef `useLocale()` skilar `'is'` virkar þetta, en ef það skilar `'is-IS'` (sem next-intl getur gert við certain configs) fer það í English branch. `locale.startsWith('is')` er öruggara.
2. `toLocaleTimeString` án `hour12: false` — getur gefið AM/PM á sumum platforms/locales.

**Lausn:** Nota `formatKlTime(h.time)` fyrir tímann (þegar til, 24h UTC) og sérstakan helper fyrir dagsetninguna:

```tsx
function formatForecastDate(isoString: string, isIcelandic: boolean): string {
  return new Intl.DateTimeFormat(isIcelandic ? 'is-IS' : 'en-GB', {
    weekday: 'short', day: 'numeric', month: 'short',
    timeZone: 'Atlantic/Reykjavik',
  }).format(new Date(isoString))
}
```

Þetta er einfaldast og þarf ekki nýjan helper file — setja beint í FerdalagidClient.tsx sem local function.
