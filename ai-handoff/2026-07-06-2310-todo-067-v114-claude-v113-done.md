## What was done

Four themes from v113: simplified time step, threshold controls (backend + UI), assumption rows fully clickable, result actions at top.

### 1. Simplified time step

Replaced three separate time inputs (`departureAt`, `latestArrivalBy`, `latestHomeBy`) with one `TimeConstraintKind` selector and one datetime input.

- Segmented control with three options: `Brottför` / `Koma fyrir` / `Heim fyrir`
- One `datetime-local` input whose label adapts to the selected kind
- API request sends exactly one of `earliestDepartureAt`, `latestArrivalBy`, `latestHomeBy` depending on selection
- Old state vars removed from `FerdalagidClient.tsx`

### 2. Threshold controls

**`lib/weather/types.ts`:**
- Added `TravelThresholdOverrides` and `ResolvedTravelThresholds` types
- Added `thresholdsUsed?: ResolvedTravelThresholds` to `TravelPlan`

**`lib/weather/thresholds.ts`:**
- Added `resolveThresholds(trailerKind, overrides?)` — merges trailer-kind defaults with user overrides
- Updated `deriveThreshold` to accept optional `resolved?: ResolvedTravelThresholds` — when provided, uses override values for threshold display in issue cards

**`lib/weather/travel.ts`:**
- `evalDrivingLeg` now accepts `ResolvedTravelThresholds` instead of reading `WEATHER_THRESHOLDS` globals
- `evaluateCandidate`, `generateCandidates`, `buildSingleDepartureTimeline` all receive + pass `thresholds`
- `candidateSeverity` and `worstCandidateOf` use `thresholds.redGustMs` for tie-breaking
- `buildHighlightedIssue` uses `thresholds.redGustMs` for gust vs wind decision, passes to `deriveThreshold`
- `buildRouteWeatherPoints` uses `thresholds.redGustMs` for `decisiveMetric`
- `checkTravelWeather` calls `resolveThresholds` once at the top, threads `resolved` everywhere, stores as `travelPlan.thresholdsUsed`
- `TravelWeatherInput` has new `thresholdOverrides?: TravelThresholdOverrides` field

**`app/api/teskeid/weather/travel/route.ts`:**
- Added `validateThresholdOverrides` helper with range checks (wind 0–40, gust 0–50, precip 0–20)
- Returns 400 `thresholds_invalid` on invalid values, not silent fallback
- Passes validated overrides to `checkTravelWeather`

**`FerdalagidClient.tsx` — threshold step:**
- New `'thresholds'` WizardStep between `'trailer'` and `'result'` in `STEP_ORDER`
- 4 number inputs with unit labels, initialized from current effective thresholds when entering the step
- "Nota sjálfgefin viðmið" resets draft to trailer defaults
- "Reikna ferðina" parses + validates, builds `TravelThresholdOverrides` (only values differing from defaults), updates state, calls `handleSubmit(overrides)` directly to avoid state timing issue
- Active thresholds shown in result card when custom overrides are in use

### 3. AssumptionRow fully clickable

`AssumptionRow` is now a single full-width `<button>` (not a card with a nested button). `aria-label` = "Breyta: {label}". Min height 44px for touch target. `editLabel` still shown as trailing text.

### 4. Result actions at top

`Breyta forsendum` + `Byrja aftur` moved to the top of the result step, rendered before the result card (after the route summary). Old bottom duplicate removed.

### 5. Assumptions step simplified

Now 5 rows instead of 6:
- From
- To
- Tímasetning (single time row: shows kind + value)
- Eftirvagn
- Veðurmörk (shows "Sjálfgefin viðmið" or custom summary)

Clicking the Veðurmörk row navigates to the thresholds step with `returnToStep = 'assumptions'`.

### 6. i18n keys added

Both `messages/is.json` and `messages/en.json` received 14 new keys under `ferdalagid`:
`stepTimesConstraintTitle`, `timeKindDeparture/Arrival/Home`, `timeLabelDeparture/Arrival/Home`, `assumptionTime`, `stepThresholdsTitle`, `thresholdsSubtitle`, `thresholdCautionWind/RedWind/RedGust/CautionPrecip`, `thresholdReset`, `thresholdSubmit`, `assumptionThresholds`, `thresholdsDefault`, `thresholdsCustom`, `thresholdsUsedLabel`, `thresholdValidationError`.

### 7. Tests added

8 new tests in `describe('threshold overrides')`:

2. `resolveThresholds` returns caravan defaults for non-none trailer
