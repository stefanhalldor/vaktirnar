## Proposal 2 - #28 overdue display: date in warning color, not combined text

Current LoanCard (line 182):

```tsx
{overdue ? t('overdue') : formatDate(item.due_at)}
```

When overdue, the actual due date disappears and only `Komið fram yfir skiladag` shows.
The plan already flags this as a loss of information and mentions the option:

> `Skila fyrir 9. júní 2026 · komið fram yfir skiladag`

I'd prefer to not go that route. The combined string is long (potentially wraps on 360 px)
and the dot-separator pattern doesn't match the rest of LoanCard's style.

Proposed rule:

- Always show the date: `dueAtFull` = `Skila fyrir {date}` (using `buildDateString` for full month)
- When overdue: apply `text-amber-600 font-medium` + `AlertTriangle` icon (already present)
- Remove the separate `t('overdue')` path for due_at display

`t('overdue')` still lives in messages for use in RecentSection (it already uses it).
The translation key is not deleted, just no longer used in LoanCard for the due line.

Result on mobile (overdue):
```
⚠ Skila fyrir 4. júní 2026
```

Result on mobile (not overdue):
```
Skila fyrir 30. júní 2026
```

This is shorter, always informative, and matches the vertical stack layout the plan wants.

If Codex or Stebbi prefer to keep `Komið fram yfir skiladag` text visible, an acceptable
compromise is a second line: `Komið fram yfir skiladag` in amber below the date line.
I'd just prefer not to combine them on the same line.

## Proposal 3 - #31 open-loans sort: flag `due_at` asc as alternative default

The plan specifies `loaned_at` as the default sort for `Enn í láni`.

An alternative: sort by `due_at` ascending (nulls last), then `loaned_at` descending
as tie-breaker. This puts items with the nearest deadline at the top, surfacing
urgency rather than recency.

Trade-offs:

| Default sort | Pros | Cons |
|---|---|---|
| `loaned_at` desc | Newest loan is on top, predictable | Overdue item buried under recent ones |
| `due_at` asc, nulls last | Urgent items surface | Items with no due date sort unpredictably |

Both are defensible. I have a slight preference for `due_at` asc, nulls last for "Enn í
láni" specifically. For `Skilað`, the plan's `returned_at` sort is clearly right.

This is a question for Stebbi/Codex. I'll implement whatever is decided. Default to
the plan's `loaned_at` unless told otherwise.

## Proposal 4 - #31 sort control: use `<select>` not a toggle button

The plan says "A small select or compact button is enough." I'd be specific: use a
native `<select>` with two `<option>` values.

Reasons:

- Mobile keyboard/accessibility: `<select>` opens the native OS picker on iOS/Android,
  no custom dropdown needed.
- No extra state management for "selected button" vs "unselected button" toggle.
- `<label>` pattern is naturally composable with `<select>`.
- Consistent with `LoanDateField`'s "native-is-fine-on-mobile" philosophy.

Suggested element:

```tsx
<label className="flex items-center gap-2 text-xs text-[#72796e]">
  <span>{t('sortLabel')}</span>
  <select value={sort} onChange={...} className="...">
    <option value="newest">{t('sortNewest')}</option>
    <option value="oldest">{t('sortOldest')}</option>
  </select>
</label>
```

If Codex prefers a toggle button for aesthetic reasons, that is fine too. I'll implement
whatever is confirmed.
